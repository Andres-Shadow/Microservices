# BDD

Tiene como objetivo:

- Orientado al comportamiento de como quiero que funcione el software

- Uso de un lenguaje común entre los involucrados (ténicos y no técnicos)

- Trabajo colaborativa

En la industria actualmente se hacen iteraciones que generalmente no son probados por lo que la puesta en marcha no es realmente el que fue probado pues la sección de pruebas siempre va una iteración atrasada.

Una de los principales objetivos es crear una comunicación efectiva entre desarrollo y pruebas



## Gherkin

Es un lenguaje usado para describir el comportamiento del sistema, cuenta con estructura y sintaxis.

Elementos básicos:

- Given 

- When

- Then

Trata de establecer un punto intermedio entre las diferentes partes, cada archivo es un feature que contiene una funcionalidad, se establece un grupo de escenarios con la finalidad de obtener resultados esperados. Cada escenario esta asociado a un criterio de aceptación.
